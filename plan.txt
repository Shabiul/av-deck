Goal
Convert the current single-page scrolling site into a clean, straightforward multi-page website while keeping the existing dark cinematic mood, colors, and premium feel. Add a light "Indian brand" touch, remove the News section, split the gallery into Images and Videos, and strengthen the proof-of-work story with reviews.
Current State

Single page (src/app/page.tsx, ~393 lines) with everything: 4 stacked hero videos, about, rental, sales, services, news, gallery, testimonials, contact.
6 service detail pages already exist at /services/[slug] (audio, lighting, video, trussing, consultancy, installations) with rich content — but styled with inline styles, inconsistent with the rest.
Theme in globals.css: dark navy (#060d18), crimson #cc0000, orange/saffron #e8930c, blue #1a6dff, Playfair Display headings + Inter body, GSAP/ScrollTrigger + Ukiyo parallax.
Assets: 17 gallery images, 16 videos in /public/video, 3 logo files.
No shared components — header/footer are duplicated inline.

Shared Components (new src/components/)
1. SiteHeader.tsx — used on every page

Logo on top, centered, bigger (~80–90px height, vs current 60px) with the nav menu bar directly beneath it — matching the requested hierarchy.
Centered nav menu, bolder & more visible: larger font size (~0.95rem), heavier weight, brighter default color (near-white instead of muted), saffron/orange (--orange: #e8930c) active state and hover underline accent — matching existing palette.
Active page highlighted based on current route (usePathname).
Compact scrolled state (logo shrinks slightly, solid dark background) so it never collides with content.
Mobile: hamburger opens a clean full-screen menu; body scroll locked while open (fixes interruption/collision issues).
Social icons kept, moved into a subtle top-right corner strip (desktop) / inside mobile menu.

2. SiteFooter.tsx — used on every page

Same as current footer but with updated nav links (real page routes), news removed, and a small "Proudly Indian · Bengaluru · Since 1992" line with a thin tricolor accent rule (subtle Indian touch).

3. PageHero.tsx

Reusable page header: background image/video + gradient overlay + eyebrow + page title. Consistent top spacing below the fixed header — guarantees no header/content collision on any page.

4. MediaLightbox.tsx

Shared lightbox for images AND videos: dark overlay, large media, caption bar showing name/tools/service label, prev/next arrows, keyboard support (Esc/arrows), body scroll locked when open.
For videos: renders a <video controls> player, preload="metadata", poster image, pauses on close.

About (/about/page.tsx) — new page

PageHero (why-video.mp4 background).

Gallery (/gallery/page.tsx) — new page with 2 sections

Tabs: "Images" | "Videos" (URL-synced so it can be linked directly).
Images tab: grid of the 17 gallery images. Hover shows the name label overlay; click/Enter opens MediaLightbox with the image full-size + name caption (e.g. "LED Wall — Corporate Launch"). Example labels written by me; structured as a simple data array so the user can easily update names later.
Videos tab: grid of the 16 existing videos shown as poster thumbnails with a play badge (no autoplaying grid = optimized). Click opens the lightbox with a <video controls> player, preload="metadata" for fast page load. Same caption format.
Media data lives in one file src/lib/media.ts (title + optional description per item) — easy for the user to edit later.
